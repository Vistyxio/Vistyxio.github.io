使用VSCode书写Markdown并上传至GitHub

可以在此下载VS Code，下载安装后打开VS Code，点击左侧的扩展图标，输入Chinese搜索并安装简体中文语言包，输入Markdown Preview Enhanced搜索并安装Markdown增强扩展。

上图中1为扩展图标；2为搜索扩展处；3为简体中文语言包，使用方法：通过使用“Configure Display Language”命令显式设置 VS Code 显示语言，可以替代默认 UI 语言。 按下“Ctrl+Shift+P”组合键以显示“命令面板”，然后键入“display”以筛选并显示“Configure Display Language”命令。按“Enter”，然后会按区域设置显示安装的语言列表，并突出显示当前语言设置。选择另一个“语言”以切换 UI 语言。；4为Markdown增强扩展。

新建一个文件夹用于存放Markdown文档，称为“工作区”，使用VS Code打开此工作区，新建文件，输入文件名（*.md）即可书写Markdown。

点此可以打开侧边预览，实时预览生成的Markdown文档。

书写过程中可以使用Ctrl+S保存，如果不需要直接显示为网页（即不需要HTML格式），可以直接将md文档上传至GitHub的仓库中，GitHub中可以直接预览Markdown。

Add file中，可以创建新文件，也可以直接上传文件。

上传GitHub后Markdown中的图片处理办法：

1. 将图片一起上传至GitHub，修改Markdown中的图片路径。（缺点：GitHub图片被墙，一般无法显示，可善用Steamcommunity 302。）

2. 将图片转为base64编码，修改Markdown中的图片格式。（缺点：base64编码过长，影响Markdown书写，可将base64编码全放在Markdown文档最后，间接引用编码。）

以下为目前在GitHub中展示HTML内容的方法，在师兄创建组织并规范工作流程后可能会发生变化。

书写完成后可以右键单击右侧预览区域，点击HTML->HTML (cdn hosted)，即可在工作区中生成与Markdown同名的HTML文档。

注册或登录GitHub，参考此篇教程创建个人站点。将生成的HTML上传至个人站点，即可通过个人站点的网址访问HTML。

例如：

https://github.com/Vistyxio/Vistyxio.github.io

https://vistyxio.github.io/HilbertTransform

https://vistyxio.github.io/PhaseSpaceReconstruction
